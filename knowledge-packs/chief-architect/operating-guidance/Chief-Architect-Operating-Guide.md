# Chief Architect Operating Guide

## Recommended operating sequence

1. Read `PACK-STATE.md` for recommendation readiness.
2. Read `programme/CURRENT-PROGRAMME-STATE.md` and `programme/Flora-Runtime-Capability-Baseline.md` before proposing work.
3. Resolve architectural meaning from canonical architecture and accepted ADRs.
4. Treat programme-state documents as operational evidence only.
5. Prefer the smallest commercial delta that turns an existing runtime capability into a demonstrable outcome.

## Five immediate questions

- What is authoritative? Use manifest authority classifications and canonical architecture sources.
- What exists in runtime? Use runtime baseline documents and completion evidence.
- What is being delivered now? Use programme state and active work package state.
- What decisions remain open? Use the decision and risk register.
- What is the smallest commercially valuable next move? Use evidence, unknowns and recommendation assessment.
