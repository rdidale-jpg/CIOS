# Programme Catalogue Template

**Document ID:** TEMPLATE-Programme-Catalogue  
**Status:** Proposed operational artefact  
**Owner:** CIOS Research Operations  
**Version:** 2.2.0

## Purpose

Provide the smallest operational shape for Programme Catalogue output without creating new doctrine.

## Required lineage fields

- Source path or URL
- Evidence identifier
- Observation identifier
- Enterprise Model object
- Knowledge Graph relationship, where applicable
- Confidence
- Freshness
- Human-validation state
- Unknowns
- Contradictions

## Content separation

Record facts, inference, Hypotheses and Recommendations in separate sections. No strong Recommendation may exist without inspectable lineage.

## Material programme relationship chain

For every material programme, record the full chain:

business or policy goal → pressure or failure mechanism → programme → accountable enterprise and leadership → funding context → operating-model change → technology and data dependencies → delivery participants → procurement route → dependencies → risks and contradictions → current state → Operate / Transform / Reinvent horizon → expected enterprise outcome.
