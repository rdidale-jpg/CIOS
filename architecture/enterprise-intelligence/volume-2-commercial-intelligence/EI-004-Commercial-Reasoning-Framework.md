# EI-004 — Commercial Reasoning Framework

**Purpose:** Define how CIOS reasons commercially about an enterprise.  
**Status:** Draft  
**Owner:** Rob / CIOS  
**Last updated:** 2026-07-03

## Architectural position

Enterprise Intelligence defines what CIOS knows about an enterprise. CIRM defines how CIOS reasons over evidence and signals. Flora is the first runtime implementation that combines both into operational intelligence.

## Purpose

Define how CIOS reasons commercially about an enterprise. This paper is documentation-only and avoids runtime implementation detail.

## Why commercial reasoning exists

Why commercial reasoning exists describes the architectural expectations for this part of the Enterprise Intelligence model. It should be evidence-linked, freshness-aware and distinct from CIRM process mechanics. CIRM explains how CIOS reasons; this section defines what CIOS should know about the enterprise.

## Core reasoning questions

- What changed?
- Why did it change?
- What does it reveal?
- What pressure does it create?
- Who owns the issue?
- Where is money likely to move?
- Where is operating pain likely to exist?
- Which transformation becomes more likely?
- Which supplier or technology position changes?
- What would confirm this?
- What would disprove this?
- What should we learn next?

## Reasoning patterns

- Revenue pressure → cost transformation
- Margin pressure → operating model change
- Capex increase → infrastructure transformation
- Capex constraint → efficiency/automation
- New CIO → technology strategy review
- New CFO → cost discipline / capital allocation review
- Regulatory pressure → compliance transformation
- Customer service decline → digital/customer transformation
- Supplier contract expiry → opportunity window
- Hiring spike → programme formation
- Hiring freeze → automation / outsourcing pressure
- Major cyber incident → secure-by-design transformation
