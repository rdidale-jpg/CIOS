# OPI-001 — Opportunity Positioning Intelligence

**Document class:** Review method
**Status:** Review
**Authority:** proposed method / proposed operational contract
**Owner:** Rob / CIOS
**Last updated:** 2026-07-14
**Production behaviour:** documentation-only
**Release-profile membership:** none — excluded from production Researcher and Assurance profiles

## Purpose

OPI-001 defines the current Review method for turning governed opportunity intelligence into candidate positioning objects. It does not change runtime behaviour and does not add any material to production Researcher or Assurance profiles.

## Scope

OPI-001 operates after opportunity discovery, research object validation and Research-to-Positioning handover. It helps positioning work reason from evidence without treating positioning language as accepted opportunity truth.

## Candidate positioning objects

OPI-001 may create candidate positioning objects for an Opportunity Twin, including:

- candidate positioning statement;
- target buyer or stakeholder frame;
- problem narrative;
- value hypothesis;
- differentiated angle;
- evidence-backed proof points;
- evidence demands;
- objections, risks, Unknowns and Contradictions;
- next learning action.

Each object must carry status, lineage, confidence, freshness and unresolved evidence demand where applicable.

## Boundary rules

OPI-001 must not:

- mark Review positioning objects as Accepted;
- override EI-006 ownership of Opportunity Prediction;
- bypass OT-001 validation boundaries;
- consume research outputs that have not passed the RTP-001 handover gate;
- add itself or its outputs to production Researcher or Assurance profiles.

## Relationship to OT-001

OPI-001 contributes candidate positioning objects to OT-001 Opportunity Twins. This is an explicit Review relationship, not an Unknown relationship. The contributed objects remain Review material pending wider validation.
