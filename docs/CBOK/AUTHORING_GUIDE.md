## Purpose

This guide explains how authors create controlled CBOK documents from initial need through approval, publication and maintenance.

## Authoring Workflow

1. Define the document intent, audience, decision context and expected downstream use.
2. Select the closest template from `docs/CBOK/templates/`.
3. Assign an identifier using the CBOK Identifier Standard.
4. Complete metadata, purpose, scope, terminology and normative references before drafting claims.
5. Draft content using explicit assumptions, evidence references and confidence labels.
6. Map claims to upstream doctrine, science, engineering or evidence records.
7. Map expected downstream engineering, SDK, application, product or operating impacts.
8. Submit for review under the CBOK Review Process.
9. Update the Master Index and related traceability records when the document becomes controlled.
10. Maintain version and review history for every material change.

## Writing Principles

- Use precise language and avoid unsupported certainty.
- Distinguish observations, hypotheses, models, standards and decisions.
- Prefer stable identifiers over prose-only cross references.
- Identify unresolved questions in appendices or issue logs rather than hiding uncertainty.
- Keep implementation mandates separate from explanatory background.

## Definition of Done

A CBOK document is ready for formal review when it has a valid identifier, complete metadata, stated confidence, traceability links, engineering mapping, version history and review history.
