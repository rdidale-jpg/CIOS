## Purpose

The CBOK Authoring System defines the controlled method for creating, reviewing, publishing and maintaining Commercial Body of Knowledge documents. It standardizes document identifiers, lifecycle states, confidence ratings, traceability expectations and reusable templates.

## Scope

This documentation area applies to CBOK constitutions, scientific frameworks, standards, reference models, commercial laws, influence models, enterprise patterns, working papers, literature reviews, experiments, validation reports and architecture documents.

## Authoring System Components

- [Authoring Guide](AUTHORING_GUIDE.md) defines the end-to-end authoring workflow.
- [Identifier Standard](IDENTIFIER_STANDARD.md) defines controlled CBOK identifiers.
- [Document Lifecycle](DOCUMENT_LIFECYCLE.md) defines document states and promotion rules.
- [Scientific Confidence](SCIENTIFIC_CONFIDENCE.md) defines confidence levels for scientific claims.
- [Operational Confidence](OPERATIONAL_CONFIDENCE.md) defines confidence levels for operational use.
- [Traceability Standard](TRACEABILITY_STANDARD.md) defines required upstream and downstream links.
- [Review Process](REVIEW_PROCESS.md) defines review roles, gates and evidence.
- [Templates](templates/) provide controlled starting points for CBOK document families.

## Required CBOK Document Structure

Every CBOK template includes document metadata, purpose, scope, terminology, normative references, content sections, traceability, version history, review history, engineering mapping and appendices.

## Governance

CBOK authors must keep claims proportionate to evidence, preserve traceability identifiers, separate scientific confidence from operational confidence and update indexes whenever a controlled document is added or materially changed.
