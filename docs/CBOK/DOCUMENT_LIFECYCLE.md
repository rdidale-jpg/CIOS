## Purpose

This standard defines lifecycle states and transition requirements for CBOK documents.

## Lifecycle States

| State | Meaning |
|---|---|
| Planned | Need identified; document not yet drafted. |
| Draft | Authoring is active; content is not authoritative. |
| Candidate | Draft is complete enough for structured review. |
| Review | Formal review is underway. |
| Approved | Document is authoritative for its scope. |
| Baseline | Approved document is suitable for engineering, SDK, application or operating use. |
| Superseded | Replaced by a newer controlled document. |
| Retired | Preserved for history but no longer active. |

## Transition Controls

Promotion to Candidate requires complete required sections. Promotion to Review requires traceability and confidence labels. Promotion to Approved requires reviewer acceptance and recorded disposition. Promotion to Baseline requires explicit downstream engineering or operational mapping.

## Versioning

CBOK documents use semantic versioning. Major versions change obligations or interpretation. Minor versions add material content without breaking existing interpretation. Patch versions clarify wording, fix errors or improve formatting.
