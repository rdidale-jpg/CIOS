# UK Banking v1 Architecture Baseline

## Decision

UK Banking is baselined as the first accepted industry implementation. Evidence proves change; Observations remember change; Enterprise Models accumulate change; reports, briefings, rankings and pipelines are transient views. Living Commercial Digital Twins remain durable assets.

## Major runtime components

- `cios.applications.flora.web.app` routes the product shell and Banking routes.
- `cios.applications.flora.banking_portfolio` owns the Banking v1 projection data, pages, rankings, signals, opportunities, suppliers and competitors.
- `cios.applications.flora.enterprise_intelligence` owns runtime explanation/audit behaviour.
- `cios.applications.flora.memory` owns evidence, observations and enterprise model storage.

## Canonical

- Evidence.
- Observations.
- Enterprise Models.
- Accepted governed source metadata.

## Non-canonical

- Executive briefings.
- Rankings.
- Reinvention-pressure assessments.
- Commercial recommendations.
- Opportunity hypotheses.
- Opportunity values.
- AI-native horizons.
- Pipeline views.
- Competitor rankings.
- User feedback.
- Runtime audit events.

## Projection boundaries

Banking pages project governed intelligence into sales-director views. They must not overwrite Evidence, Observations or Enterprise Models. Opportunity estimates, recommendations, rankings and timing windows are commercial hypotheses for challenge, not Enterprise Intelligence canon.

## Banking-specific configuration

Five accepted banks, twelve industry signals, fifteen current opportunity hypotheses, Banking themes, AI-native capabilities, reinvention positions, supplier entries and competitor capability views are held in the Banking portfolio module.

## Reusable cross-industry components

The product shell, safe-unavailable pattern, evidence inspection, audit event pattern, portfolio/navigation structure, opportunity rendering and memory architecture are reusable.

## Banking-specific components

Banking PESTLE, industry forces, AI-native bank reference model, UK Banking timeline, bank priority ranking, bank theme scores, supplier/competitor views and Banking opportunity hypotheses are Banking-specific.

## Assumptions and boundaries

Recommendation and valuation outputs are inspectable but non-canonical. Lineage keys and source labels remain visible. Runtime audit events are retained for inspection but do not become canonical Enterprise Models. Safe-unavailable behaviour fails closed for unknown banks. Determinism is achieved by fixed accepted inputs and sorted JSON snapshots. Human-supplied knowledge remains labelled and requires validation before calibration changes.
