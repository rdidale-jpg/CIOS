# Founder Knowledge

Founder Knowledge preserves strategic reasoning, future-state design, commercial hypotheses, enterprise reinvention methods, recovered blueprints and supporting design evidence.

It is distinct from public Evidence, Enterprise Intelligence, CIOS architecture, runtime configuration and operational commissions. Founder Knowledge is governed preservation material: it provides provenance and context without becoming evidence, architecture or an operational instruction by default.

## Authority Rules

- Founder Knowledge may inform CIOS and Flora.
- It does not override Accepted ADRs or owning architecture papers.
- It must not be presented as public-domain evidence.
- It is excluded from production Researcher, Assurance, Reviewer and architecture-authority profiles unless explicitly commissioned for a governed task.
